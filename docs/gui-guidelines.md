# UNVET GUI 運用方針 (v1)

## 1. 目的

本ドキュメントは、UNVET の GUI を「目立たず堅実に運用するための補助UI」として定義し、
実装範囲と優先順位を明確化する。

## 2. 採用アーキテクチャ

- GUI 技術: Tauri (WebView ベース)
- コア処理: Rust 側で完結 (入力受信 / 正規化 / キャリブレーション / マッピング / 出力)
- 役割分離:
  - Rust: データプレーン (低遅延・高安定)
  - GUI: コントロールプレーン (状態表示・設定変更)

## 3. GUI スコープ (v1)

### 3.1 必須

- 出力 ON/OFF
- 一時停止
- 入力ソース切替 (UDP / TCP)
- 出力方式切替 (ETS2 / Mouse / Keyboard)
- 再キャリブレーション
- 接続状態と現在値表示 (yaw / pitch / confidence)

### 3.2 優先度高

- 感度 (左右 / 上下)
- デッドゾーン
- 平滑化係数

### 3.3 GUI 非対象 (設定ファイル管理)

- すべての詳細設定
- 実験パラメータ
- 運用上ほぼ変更しない内部しきい値

## 4. 簡易プレビュー方針

- レティクル表示:
  - 画面枠に対する注視位置の簡易表示
- 矢印表示:
  - 移動量出力モードでは、中心からのベクトル方向・大きさを表示
- 状態色:
  - active / low-confidence / paused を視覚的に区別

## 5. トレイ方針 (Windows)

- 初期優先度: 中
- 可能なら実装:
  - 最小化時にタスクトレイ格納
  - トレイから再表示 / 一時停止 / 終了
- 難易度が高い場合は v1 必須にしない

## 6. 性能・更新制約

- 制御ループと GUI 更新を分離する
- GUI 更新頻度は制限する (目安: 10-30Hz)
- GUI 停止時もコア処理は継続できる構造を維持する

## 7. 実装順序 (推奨)

1. 最小 UI (必須操作のみ)
2. 調整 UI (感度・デッドゾーン・平滑化)
3. 簡易プレビュー (レティクル / 矢印)
4. トレイ連携 (可能なら)
